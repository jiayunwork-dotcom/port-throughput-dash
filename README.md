# 港口集装箱吞吐量预测与堆场利用率优化分析平台

基于 **Python + Dash + Plotly** 构建的港口运营智能分析面板，支持吞吐量时序预测、堆场热力图分析、翻箱率蒙特卡洛模拟、KPI仪表板、What-if情景分析和PDF报告导出。

---

## ✨ 功能模块

### 1. 数据导入模块
- **三类CSV文件支持**：
  - 船舶靠泊记录（船名、到港时间、离港时间、载箱量TEU、分配泊位编号）
  - 集装箱流转记录（箱号、进场时间、出场时间、堆场区域、贝位-列-层坐标、箱型20/40尺、状态）
  - 堆场布局定义（区域编号、贝位数量、列数、最大堆叠层数）
- 内置模拟数据生成器，一键生成约10万条集装箱流转记录和300条船舶靠泊数据
- 支持拖拽上传自定义数据文件

### 2. 吞吐量时序预测模块
- **三种算法可切换对比**：
  - **ARIMA**：自动定阶（AIC准则），7天季节性周期
  - **Prophet**：节假日效应 + 周周期 + 年周期
  - **简化LSTM**：单层128单元，滑动窗口30天预测未来7天
- 可视化输出：
  - 实际值 vs 预测值折线叠加
  - MAPE和RMSE指标卡片（自动着色告警）
  - 残差分布直方图（带正态拟合曲线）
  - 未来7/14/30天80%置信区间阴影带
- 支持预测粒度（日/周）和历史训练长度（30/60/90天）切换

### 3. 堆场利用率分析
- **俯视热力图**：贝位×列矩阵，颜色深浅表示堆叠高度
- **时间轴滑块**：从0%到100%回放任意时刻的堆场状态快照
- **多区域利用率趋势**：各区域占用率折线 + 85%高负荷预警线
- **超期箱统计**：
  - 可配置免费堆存期（默认5天）
  - 阶梯费率滞箱费计算：
    - 第6~10天：50元/天
    - 第11~20天：80元/天
    - 超过20天：120元/天
  - 按客户/航线/区域分组统计，TOP10客户排行

### 4. 翻箱率蒙特卡洛模拟
- **三种堆存策略**：
  - **策略A**：随机堆放（基准）
  - **策略B**：按预计提箱时间排序（早提放上层）
  - **策略C**：按航线聚堆（同航线箱放同列）
- 支持500/1000/2000次模拟，可配置堆场填充率
- 输出：翻箱率均值、P95百分位数、策略对比柱状图

### 5. KPI运营指标面板
- **四大核心KPI**：
  - 桥吊效率（TEU/台时）
  - 泊位利用率（占用小时/可用小时）
  - 集卡周转时间（进闸→堆场落地）
  - 船时效率（载箱量TEU/靠泊时长）
- 支持日/周/月聚合
- 连续3天低于均值-1标准差自动标红告警

### 6. What-if情景分析
- **情景1**：吞吐量增长（+10%/20%/30%）→ 堆场峰值利用率影响
- **情景2**：新增堆场区域（可配置贝位×列×层）→ 利用率下降幅度
- **情景3**：缩短平均堆存天数（-1/2/3天）→ 超期箱和翻箱率改善
- 一键运行全部情景，对比柱状图+结果表格

### 7. PDF报告导出
- 七章结构化报告，含：
  - 吞吐量统计概览
  - 三种算法预测准确度评估
  - 堆场利用率峰谷时段分析
  - 翻箱率策略对比结论
  - 超期箱TOP10客户清单
  - KPI仪表盘（截图+表格）
  - What-if分析结论摘要

## 🚀 快速开始

### 方式一：一键启动脚本（推荐）

**macOS/Linux**：
```bash
chmod +x run.sh
./run.sh
```

**Windows**：
```cmd
run.bat
```

脚本会自动：
1. 创建Python虚拟环境 `.venv`
2. 安装所有依赖包
3. 生成模拟数据（首次运行）
4. 启动Dash应用服务器

### 方式二：手动启动

```bash
# 1. 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 生成模拟数据（可选，首次运行自动生成）
python -c "from src.data_generator import PortDataGenerator; PortDataGenerator().generate_all('data')"

# 4. 启动应用
python app.py
```

启动后访问：**http://localhost:8050**

## ⚠️ 注意事项

1. **Prophet依赖**：Prophet需要`pystan`或`cmdstanpy`，安装可能较慢，建议使用conda安装
2. **PyTorch**：如无需LSTM预测，可跳过PyTorch安装，面板支持ARIMA和Prophet降级使用
3. **内存**：15万条集装箱数据+LSTM训练建议≥8GB内存
4. **端口冲突**：默认端口8050，可在`app.py`底部修改`APP_PORT`

## 🎓 使用流程建议

1. **启动应用** → 查看自动生成的模拟数据
2. **熟悉面板** → 依次浏览概览→预测→堆场→翻箱→KPI→What-if各模块
3. **运行预测** → 选择三种算法对比，查看各指标优劣
4. **翻箱模拟** → 调整填充率和模拟次数，对比策略B/C相对策略A的改善
5. **What-if分析** → 点击"一键运行全部"，查看各情景关键指标变化
6. **导出报告** → 点击导航栏"工具→导出PDF报告"，生成完整分析文档
7. **上传真实数据** → 使用数据导入区拖拽自己的CSV文件替换模拟数据
